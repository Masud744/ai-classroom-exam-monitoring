Project Title: AI Classroom and Exam Monitoring System (Full Stack AI Platform)

We are planning to build a complete AI-based classroom and online exam monitoring platform that includes a local AI monitoring client, backend server, database, authentication system, and web dashboards for students and teachers.

This project is not just a computer vision script; it is a full system consisting of multiple components working together.

System Overview:
The system will monitor students during online classes or exams using a camera and computer vision. The AI system will detect behaviors such as face presence, multiple faces, eye direction, head pose, sleeping (eyes closed), talking (mouth movement), and phone detection. Based on these detections, the system will calculate an Attention Score (focus level) and a Suspicious Score (cheating probability). These scores and behavior logs will be sent to a backend server and stored in a database. Teachers will be able to monitor students live and view reports, while students will be able to see their own results.

System Architecture:
Student Website (Login / Start Exam)
↓
Download / Run Monitoring Client App (.exe)
↓
AI Monitoring Client (Python + OpenCV + MediaPipe + YOLO)
↓
Send Monitoring Data → FastAPI Backend (Render)
↓
Supabase Database (PostgreSQL)
↓
Web Dashboard (Netlify)
↓
Teacher Monitoring / Reports / Analytics

Main System Components:

1. Monitoring Client Application (runs locally on student PC)
2. AI Monitoring System (computer vision + behavior detection)
3. FastAPI Backend Server
4. Supabase Database
5. Authentication System (Student & Teacher login)
6. Student Dashboard
7. Teacher Dashboard
8. Reports and Charts
9. Deployment (Render + Netlify)
10. GitHub for source code and client distribution

AI Monitoring Features:

* Face Detection
* Face Count (multiple face detection)
* Face Landmarks (MediaPipe Face Mesh)
* Eye Tracking (eye direction)
* Head Pose Detection (looking left/right/up/down)
* Sleep Detection (eyes closed using EAR)
* Mouth/Talking Detection (using MAR)
* Phone Detection (YOLO object detection)
* Attention Score Calculation
* Suspicious Score Calculation
* Behavior Status Display
* Periodic Log Sending to Backend

User Roles:

1. Student:

   * Sign up / Login
   * Start exam
   * Run monitoring client
   * View own attention score
   * View own suspicious score
   * View history and reports
   * Cannot see other students

2. Teacher/Admin:

   * Login
   * View all students
   * Live monitoring dashboard
   * Suspicious alerts
   * View reports and analytics
   * See charts (attention vs time, suspicious vs time)
